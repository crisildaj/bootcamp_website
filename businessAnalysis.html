Process Mapping is a method used to outline the steps within a process, identifying task ownders and detailing expected timelines

Despite Process Mapping not being used, it would have been useful for:

  • Outlining the steps in creating the website that the Technophobes have created.
  • There is a start and end point to every process map and specific tasks are to be completed before proceeding to the next.
  • The process map on the right was done in a exercise during the business analysis week and is for the graduate process for the technology graduate scheme.
  • If there is a failure at any point within the process,a notification is sent tothe applicant to apply next year. If a pass , then proceed to the next step.

The key for the Process Map is as follows

  • Initial Nodes and Final Nodes determine the start and the end to the process. Each task is shown with the rectangle and when completed follows the next recttangle connected via a flow arrow.
  • The decision diamons determine a pass or fail condition, where if met, proceed to the pass/yes flow arrow, else proceed the the fail/no path.
  • Joins can only be proceeded when both events are completed at the same time, Forks occur when a individual event can lead to two different events.
  • Applying Swimlanes means that the process is split horizontallly to different lanes, where each lane represents a actor doing a specific task within the process map
